# 🚀 Hugging Face Deployment Guide for Companies

This guide details the process of deploying the **Agentic Hiring Suite** to a new Hugging Face Space.

## 1. Google Cloud Setup (One-Time)
- Go to [Google Cloud Console](https://console.cloud.google.com/).
- Create a project and enable the **Gmail API**.
- Configure **OAuth Consent Screen** (Internal or External).
- Add Scopes: `gmail.readonly` and `gmail.send`.
- Create **OAuth Client ID** (Web Application).
- **CRITICAL:** Add this Redirect URI:
  `https://[username]-[space-name].hf.space/auth/gmail/callback`
- Download the JSON and save it as `Backend/client_secret.json`.

## 2. Hugging Face Space Creation
- Create a new Space.
- **SDK:** Docker (Blank).
- **IMPORTANT:** Do NOT add a variable named `SPACE_ID`. Hugging Face provides this automatically.

## 3. Configuration (Settings > Variables and Secrets)
Add these strictly:

### Secrets
- `OPENAI_API_KEY`: Your production API key.
- `GROQ_API_KEY`: Your Groq API key.

### Variables
- `BASE_URL`: `https://[username]-[space-name].hf.space`
- `SMTP_SERVER`: `smtp.gmail.com` (If not using Gmail API)
- `SMTP_PORT`: `587`

## 4. Deployment Commands
Run these from your local terminal:

```bash
# Add the HF remote (replace with your token and space details)
git remote add huggingface https://[username]:[token]@huggingface.co/spaces/[username]/[space-name]

# Push code (Use --force if remote has divergent history like README changes)
git add .
git commit -m "Deployment Push"
git push huggingface main --force
```
